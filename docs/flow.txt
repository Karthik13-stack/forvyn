📘 LexiLaw – Strict RAG Pipeline (README)
Overview

LexiLaw is a legal AI system built using a Retrieval-Augmented Generation (RAG) architecture.
It performs clause-level legal analysis, rewriting, and document generation using verified legal context only.

This repository implements strict grounding guarantees, ensuring the LLM cannot hallucinate legal context when retrieval fails.

🧠 Architecture Summary

LexiLaw follows a hard-gated RAG pipeline:

User Query / Clause
        ↓
Clause Chunking
        ↓
Embedding Generation
        ↓
Vector Store (FAISS)
        ↓
Retriever
        ↓
[ HARD GATE ]
        ↓
Prompt Builder
        ↓
LLM (Gemini)


Critical Rule:
If retrieval returns zero context, the LLM is never called.

Status: ✅ Implemented & working

FAISS index built from embedded legal clauses

Supports semantic similarity search

Metadata preserved where required

Status: ✅ Implemented & working

4️⃣ Retrieval Layer

Query embeddings matched against FAISS index

Top-k relevant legal chunks returned

Status: ✅ Implemented & working

5️⃣ 🚨 Strict Retrieval Hard Gate (MOST IMPORTANT FIX)

A hard gate is enforced at the orchestration layer:

If retrieved_chunks == []

❌ Prompt is NOT built

❌ LLM is NOT called

✅ System returns: "NO CONTEXT RETRIEVED"

This prevents:

Hallucinated legal reasoning

Fabricated “Quoted Context”

Unsafe legal answers

Status: ✅ Implemented in:

RAGGenerationFlow

ClauseRegenerationFlow

6️⃣ Prompt Builder (Grounded Only)

Prompts are constructed only from retrieved context

Context is injected verbatim

LLM is explicitly constrained to provided text

Status: ✅ Implemented & safe

7️⃣ LLM Invocation (Controlled)

LLM is invoked only after retrieval passes

Request tracing via request_id

No fallback reasoning when context is missing

Status: ✅ Implemented & controlled
